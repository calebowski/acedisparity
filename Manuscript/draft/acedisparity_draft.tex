\begin{abstract}
\noindent
Ancestral state estimation is a widely used method to infer unobserved traits across a phylogeny, extending analyses beyond incomplete fossil records. 
In studies of morphological disparity ancestral state estimation can account for ghost lineages, yet estimates are often confounded with high uncertainty. 
How best to deal with this uncertainty remains unclear. 
Using simulations, we compared pre- and post-ordination estimation methods, as well as both point estimate and probabilistic approaches for treating uncertainty, under continuous and discrete trait models and varying fossil sampling levels.
The relative disparity error was calculated from the difference between our estimated disparity values and the true disparity values from the original simulations.
We then extended this framework to evaluate the recovery of changes in disparity across time by simulating mass extinction events with varying intensity and selectivity.
We find that pre-ordination probabilistic estimations outperform post-ordination and point estimation methods and that omission of ancestral state estimations leads to underestimates of disparity.
As expected, higher fossil sampling improves estimation accuracy, although recovered disparity accuracy was robust even at low fossil sampling. 
Similar patterns were observed when recovering changes in disparity across mass extinction events but declined significantly when fossil sampling was low, highlighting that fossil inclusion is essential when assessing disparity in deep time.
These results demonstrate that explicitly propagating uncertainty, rather than collapsing states into point estimates, is more likely to recover accurate patterns in disparity, reinforcing the need to move beyond point estimates when working with the incomplete fossil record.
\end{abstract}

\noindent\textbf{Keywords:} ancestral state estimation; morphological disparity; fossils; simulation; uncertainty; macroevolution

\subsubsection{Trait simulation} \label{trait simulation}
For each phylogeny, we simulated discrete and continuous trait data separately.
Continuous trait data were simulated under variations of Brownian motion (BM) and Ornstein-Uhlenbeck (OU) models (table \ref{tab:models}). 
We simulated BM with variance = 0.25.
We also simulated BM with a trend ($BM_t$), whereby the ancestral trait value increased by an increment of 0.1 multiplied by branch length.
For OU, the data were simulated as strong ($OU_{st}$), weak ($OU_w$) and with differing optimum and root states ($OU_{sh}$).
Following \cite{gearty2024impact}, the alpha ($\alpha$) parameter was set based on phylogenetic half life ($\frac{\mathit{ln(2)}} {\alpha}$) to account for differing tree heights across simulations \citep{cooper2016cautionary}.
For the strong and weak OU models $\alpha$ was set as 25\% and 75\% of the tree height respectively.
We simulate 20 independent characters for each model (\textit{BM}, $BM_t$, $OU_w$, $OU_{st}$, $OU_{sh}$) across each phylogeny.

Discrete trait evolution was simulated using a discrete-time Markov process, where the probability of state transition along each branch was proportional to the product of the transition probability and branch length.
Discrete trait data were simulated using across three rate categories, representing fast (0.9), medium (0.1) and slow (0.01) transition rates (table \ref{tab:models}). 
These rates were assigned by generating equal-rate (ER) transition probability matrices.
For each category, 85 binary characters and 15 three-state characters were simulated, following the typical scoring ratio in empirical data \citep{guillerme2016effects, casali2023morphological}.

\subsection{Fossil sampling}
In order to simulate different levels of fossil sampling, extinct tips were either retained (`fossilized') or pruned using a Bernoulli sampling method at five different probabilities: 0\%, 5\%, 15\%, 50\% and 100\%. 
This method means each extinct tip has an equal probability of being `fossilized' at each sample level, irrespective of the total tree size.
The corresponding trait matrices were subset to match each downsampled tree.

\subsubsection{Pre-ordination estimation} \label{pre-ord}
Firstly, we compared pre- versus post-ordination estimation methods.
Ancestral state estimation of trait data can be carried out at two points either side of the ordination step in the disparity pipeline \citep{lloyd2018journeys}.
Pre-ordination estimation (pre-OASE) involves the estimation of the raw trait values for each node.
Post-ordination estimation (post-OASE) calculates the continuous scores across each PC axis of an ordination matrix.
%TG: then say briefly what these two methods do (for both continuous and discrete - i.e. estimating before ordination or estimating within ordination). 
The pre-OASE method for estimating ancestral discrete characters involves estimating the marginal likelihoods for each discrete state at each node, based on a Maximum-Likelihood derived estimate of the M\textit{k} transition matrix and the tree topology given \citep{pagel1999maximum,yang2006computational}.
Given that the transition rates were set \textit{a priori} during simulations, the equal-rates (ER) model was used to avoid model misspecification as a confounding factor on estimation accuracy \citep{keating2023best}.

Pre-OASE of continuous characters models evolution under a Brownian motion model, whereby ancestral state values lie within a multivariate normal distribution that is estimated given the observed tip data and the shared evolutionary history between taxa \citep{schluter1997likelihood,rohlf2001comparative}.

The width of confidence intervals reflects the uncertainty in continuous ancestral estimates, which typically increases with node depth and decreases with the number of informative descendant lineages.

For discrete characters, distance matrices were computed using the Maximum Observed Rescaled Distance metric (MORD) \citep{lloyd2016estimating} and ordinated using principal coordinates analyses (PCoA) (\texttt{cmdscale()} function). 
Continuous data were ordinated using principal components analysis (PCA) (\texttt{prcomp()} function).

\subsubsection{Post-ordination estimation} \label{post-ord}
Post-OASE involves the estimation of a node's position within a multidimensional ordinated matrix (commonly referred to as the trait space, morphospace, functional space, etc.).
The generation of an ordinated matrix differed between discrete and continuous characters.
For discrete characters, distance matrices were computed using the Maximum Observed Rescaled Distance metric (MORD) \citep{lloyd2016estimating} and ordinated using principal coordinates analyses (PCoA) (\texttt{cmdscale()} function). 
Continuous data were ordinated using principal components analysis (PCA) (\texttt{prcomp()} function).
For both the discrete- and continuous- character derived trait spaces, the ancestral states across all PC axes were then estimated under a Brownian motion model, as detailed in section \ref{pre-ord}.

\subsubsection{Uncertainty treatment} \label{uncertainty}
For each estimation method, we also compared point estimate and probabilistic approaches. 
The point estimate approach returns a fixed single value for each estimate.
For discrete characters, this involved a strict majority rule whereby the state with the highest scaled likelihood was selected at each node for each character.
In scenarios where state likelihoods are equal, one state is chosen at random.
We also employed a relative majority rule, whereby any state at a scaled likelihood greater than the maximum observed scaled likelihood minus the inverse number of possible states is collapsed, otherwise states are left as uncertain (supplementary).
For continuous characters, the point estimate returned was the median of the ancestral confidence interval. 

The probabilistic approach differs in that uncertainty in estimates are explicitly accounted for by sampling multiple results rather than a single point estimate. 
For each discrete character states at each node were sampled 100 times according to their scaled likelihoods (resulting in 100 character estimation matrices). 
In a scenario where there is uncertainty in a binary character for a specific node, e.g., 0.55 for state 0 and 0.45 for state 1, this distribution approach would results in a 55\% probability of sampling state 0 in each matrix for the respective node (and 45\% chance of sampling state 1).
Continuous characters were treated similarly, with 100 matrices replicated and ancestral state estimates for each node and character sampled across a normal distribution centered on the mean estimation and spread across the estimated confidence interval.
Both procedures are implemented using the \texttt{sample = 100} argument in the \texttt{dispRity::multi.ace} function \citep{guillerme2018disprity}.

\subsection{Disparity metrics}
To ensure fair comparison between different fossil sampling levels only the nodes present across all fossil sampling levels were retained from the ancestral state estimation outputs for each phylogeny; therefore, comparisons in disparity calculations between fossil sampling levels differed solely on the difference in number of fossil tips and the estimations of node states.
Discrete and continuous pre-ordination ancestral state estimations were ordinated in the same manner as described in section \ref{post-ord}. %TG: I think this bit should go above in the section where you talk about ordination.

When assessing overall disparity, we wanted to understand how well ancestral state estimation can recover true patterns in size and density in trait space occupancy \citep{guillerme2020shifting}.
% Since we were assessing overall disparity, position was not a metric that can be discerned due to the centring of ordinated matrices. TG: this sentence is probably not necessary. But might bring it back if a reviewer asks for why no position?
Size-based metrics describe the spread of points in the trait space \citep{guillerme2020shifting}.
To explore how well the size of the trait space was recovered, we calculated the sum of quantiles.
The sum of quantiles calculates the range on each PC axis for the 95\% of the data and then sums these ranges across all axes.
This metric has similar properties to the often used sum of ranges metric by describing the total trait space exploited \citep{foote1992rarefaction}, but with less sensitivity to outliers and is thus less sensitive to sample size.
For density, we measured the sum of variances \citep{ciampaglio2001detecting,wills2001morphological} and the average nearest neighbour distance \citep{foote1992rarefaction}. 
% The sum of variances is a very widely used metric in disparity analyses. %TG: pro intuition writing tip: if you say "the most used" I'd probably want to see a meta analysis to prove it. if you say "very widely used" I can agree with you without any extra reference/work ;)
Disparity values were calculated across each estimation and the true simulation so that a relative disparity error could be calculated.

\begin{equation}
\text{Relative disparity error} = \frac{\text{Estimated disparity} - \text{True disparity}}{\text{True disparity}}
\end{equation}

We calculated disparity across three different treatments of the ordinated matrices.
Firstly, we used the untreated ordination output to examine how well ancestral state estimation recovers the true disparity signal across all PC axes. 
Secondly, we min-max standardised the PC axes so that the absolute magnitude in PC scores did not influence the disparity metric scores (i.e. in this case, the max-score for PC1 was always standardised to 1). %TG: is that correct? or 2? (spread from [-1;1])
Thirdly, we only used the minimum number of dimensions that were contained across all fossil sampling levels for that specific phylogeny so that each fossil sampling level, i.e., the first 48, 98 and 148 dimensions for the 50, 100 and 150 living taxa simulations, respectively.
This was to negate any effect that higher-dimensional matrices were more likely to find the true disparity signal.
